# Email Design Brief — HLLC Store

เอกสารนี้ใช้ส่งให้ designer / Claude ออกแบบอีเมลใหม่ มี 3 ส่วน:
1. มีอีเมลกี่แบบ + แต่ละแบบคืออะไร + ตัวแปรที่ต้องใส่
2. โครงเลย์เอาต์ที่ใช้ร่วมกัน
3. ⚠️ ข้อจำกัดทางเทคนิค (ต้องอ่าน — อีเมลเขียนแบบเว็บปกติไม่ได้)

---

## 1. มี 5 แบบ (+1 สำรอง)

ทุกแบบส่งหาลูกค้า เนื้อหา **ภาษาไทย** (มี subtitle ภาษาอังกฤษบน header)

| # | ชื่อ | ส่งเมื่อ | Subject | สีหลัก/badge | ตัวแปรที่ใช้ |
|---|------|---------|---------|--------------|--------------|
| 1 | สลิปผ่าน | แอดมินอนุมัติสลิป | สลิปผ่านการอนุมัติแล้ว | เขียว `#16a34a` | `customerName`, `trackingUrl` |
| 2 | สลิปไม่ผ่าน | แอดมินปฏิเสธสลิป | กรุณาส่งสลิปใหม่ | แดง `#dc2626` | `customerName`, `trackingUrl`, `note?` (ข้อความจากร้าน) |
| 3 | จัดส่งสินค้า | ใส่เลขพัสดุ | จัดส่งสินค้าแล้ว | น้ำเงิน `#2563eb` | `customerName`, `trackingNumber`, `trackingUrl` |
| 4 | มารับสินค้าเอง | ออเดอร์รับเองพร้อมให้รับ | สินค้าพร้อมให้รับแล้ว | เหลือง `#92400e` | `customerName`, `location` (เช่น D1), `trackingUrl` |
| 5 | ยกเลิกคำสั่งซื้อ | แอดมินยกเลิก | คำสั่งซื้อถูกยกเลิก | แดง `#dc2626` | `customerName`, `reason` (ข้อความ/เหตุผล), `trackingUrl` |

> ลูกค้าค้นออเดอร์ด้วยเบอร์โทร → **ไม่แสดงเลขออเดอร์ในอีเมล**
> `trackingUrl` = ลิงก์หน้าติดตามคำสั่งซื้อ `https://<site>/profile?customerPhone=<phone>`
> แบบที่ **2 และ 5** มีกล่อง "ข้อความจากร้าน/เหตุผล" ที่จะมีหรือไม่มีก็ได้ (optional)

---

## 2. โครงเลย์เอาต์ (ใช้ร่วมทุกแบบ)

เรียงบนลงล่าง 1 คอลัมน์ กว้างสุด ~600px จัดกลาง:

1. **Header** — พื้นสีแบรนด์ ไล่เฉด `#85241F → #B72D2A`, โลโก้ตัวอักษร "HLLC" สีขาว + subtitle อังกฤษ
2. **Badge สถานะ** — ป้ายกลม สีตามตาราง (ตัวอักษรเข้ม + พื้นอ่อนของสีนั้น)
3. **หัวข้อใหญ่** (ไทย) + ทักทาย "สวัสดีคุณ <ชื่อ>" + ย่อหน้าอธิบายสั้นๆ
4. **กล่องข้อความจากร้าน** *(เฉพาะแบบ 2, 5 และเมื่อมีข้อความ)* — กล่องมีเส้นขอบซ้ายสีแบรนด์ + หัวข้อ + ข้อความ
5. **ตารางรายละเอียด** — แถว label/value เช่น สถานะ / เลขพัสดุ / จุดรับสินค้า
6. **ปุ่ม CTA** — "ดูคำสั่งซื้อ / ติดตามสถานะ" ลิงก์ไป `trackingUrl` (ปุ่มสีแบรนด์)
7. **Footer** — ข้อความ "ส่งอัตโนมัติ กรุณาอย่าตอบกลับ" สีเทาเล็ก

### สีแบรนด์
- Primary `#85241F` · Secondary `#B72D2A`
- เทา: หัวข้อ `#111827`, เนื้อความ `#4b5563`, รอง `#6b7280`, เส้น `#e5e7eb`, พื้นกล่อง `#f9fafb`
- สีสถานะ (พื้นอ่อน/ตัวเข้ม): เขียว `#dcfce7`/`#166534` · แดง `#fee2e2`/`#b91c1c` · น้ำเงิน `#dbeafe`/`#1d4ed8` · เหลือง `#fef3c7`/`#92400e`

## 3. ⚠️ ข้อจำกัดทางเทคนิค (สำคัญมาก — บอก designer ให้ครบ)

อีเมลไม่ใช่หน้าเว็บ ต้อง render ได้ใน Gmail / Outlook / Apple Mail ฯลฯ จึงต้อง:

- **ภาษา: HTML + inline CSS เท่านั้น** (`style="..."` บนทุก element)
  - ❌ ห้าม: ไฟล์ CSS แยก, `<style>` ใน head (บาง client ตัดทิ้ง), Tailwind/class, JavaScript, `<link>`
- **เลย์เอาต์ใช้ `<table>` เป็นหลัก** (ไม่ใช่ flexbox / grid / `position`) เพื่อรองรับ Outlook
- **ความกว้างสูงสุด ~600px** คอลัมน์เดียว จัดกลาง
- **ฟอนต์ต้องเป็น web-safe + รองรับไทย** → `font-family: Arial, 'Noto Sans Thai', sans-serif;` (โหลดฟอนต์เองไม่ได้)
- **ปุ่ม = `<a>` ทำเป็นปุ่ม** (bulletproof button) ห้ามใช้ `<button>`
- **รูปภาพมักถูกบล็อกโดยอัตโนมัติ** → โลโก้ใช้ "ตัวอักษร HLLC" ไม่พึ่งรูป ถ้าจะใช้รูปต้องเป็น URL https เต็ม + ใส่ `alt`
- **สีใส่เป็น hex inline** ห้าม CSS variables / `oklch` / `rgb()` สมัยใหม่บางตัว
- **อย่าพึ่ง dark mode** และหลีกเลี่ยงพื้นขาว-ตัวอ่อนที่อ่านไม่ออก
- **หน่วยเป็น px**, ระยะห่างใช้ `padding` บน cell (ไม่ใช่ `margin` ที่ Outlook มองไม่เห็นบางที)
- รองรับทั้งมือถือและเดสก์ท็อป (single column ปรับตัวได้เอง)

### รูปแบบผลลัพธ์ที่อยากได้
ฟังก์ชัน (TypeScript) ที่คืน **string HTML** โดยมีจุดเสียบตัวแปรตามตารางข้อ 1 — เช่น
```ts
function baseEmailHtml(opts: {
  subtitle: string; badge: string; badgeBg: string; badgeColor: string;
  headline: string; intro: string;                       // intro เป็น HTML
  detailRows: { label: string; value: string; color?: string }[];
  note?: { label: string; text: string };                // optional callout
  ctaUrl: string;
}): string
```
แล้วแต่ละแบบ (5 อัน) เรียก `baseEmailHtml` ด้วยค่าของตัวเอง — ออกแบบ **ตัว `baseEmailHtml` ตัวเดียว** ก็พอ ที่เหลือคือป้อนค่า

> โค้ดปัจจุบันอยู่ที่ [`lib/backend/email-service.ts`](../lib/backend/email-service.ts) (ดูของเดิมเป็นต้นแบบได้)
