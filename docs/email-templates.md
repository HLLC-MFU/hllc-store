# อีเมลแจ้งลูกค้า (HLLC Store)

ทุกอีเมลใช้เทมเพลตกลางตัวเดียว (`baseEmailHtml` ใน [`lib/backend/email-service.ts`](../lib/backend/email-service.ts)) โครงเหมือนกันหมด ต่างกันที่ **ป้ายสถานะ (badge) / สี / ข้อความ / รายละเอียด**

> ลูกค้าค้นหาคำสั่งซื้อด้วย**เบอร์โทร** อีเมลจึงไม่แสดงเลขออเดอร์ (ตัด `#รหัส` ออกแล้ว) ปุ่มในอีเมลพาไปหน้าติดตามโดยแนบเบอร์ให้อัตโนมัติ

## โครงหน้าตา (ทุกฉบับ)

```
┌───────────────────────────────┐
│   [ HLLC ]  ← header สีแบรนด์   │   #85241F → #B72D2A
│   subtitle (อังกฤษ)            │
├───────────────────────────────┤
│        ( badge สถานะ )         │   สีตามประเภท
│        หัวข้อใหญ่ (ไทย)         │
│   สวัสดีคุณ <ชื่อ> + คำอธิบาย     │
│                               │
│   [ กล่องข้อความจากร้าน ]       │   แสดงเฉพาะบางแบบ (ถ้ามี)
│                               │
│   ┌─ รายละเอียด ─────────┐     │
│   │ สถานะ : ...          │     │
│   └──────────────────────┘     │
│                               │
│   [ ปุ่ม: ดูคำสั่งซื้อ/ติดตาม ]   │   → /profile?customerPhone=...
├───────────────────────────────┤
│   footer: ส่งอัตโนมัติ ฯลฯ       │
└───────────────────────────────┘
```

## รายการอีเมลตามสถานะ

| สถานะ / เหตุการณ์ | ฟังก์ชัน | ส่งเมื่อ | Subject | ป้าย (badge) | สี |
|---|---|---|---|---|---|
| **สลิปผ่าน** | `slipApprovedEmail` | แอดมินกดอนุมัติสลิป | สลิปผ่านการอนุมัติแล้ว | ชำระเงินสำเร็จ | เขียว |
| **สลิปไม่ผ่าน** | `slipRejectedEmail` | แอดมินปฏิเสธสลิป | กรุณาส่งสลิปใหม่ | สลิปไม่ผ่าน | แดง |
| **จัดส่งสินค้า** | `trackingNumberEmail` | แอดมินใส่เลขพัสดุ | จัดส่งสินค้าแล้ว | จัดส่งแล้ว | น้ำเงิน |
| **มารับเอง** | `pickupReadyEmail` | ออเดอร์แบบรับเองเปลี่ยนเป็น "จัดส่ง" | สินค้าพร้อมให้รับแล้ว | พร้อมให้รับ | เหลือง |
| **ยกเลิก** | `orderCancelledEmail` | แอดมินยกเลิกออเดอร์ | คำสั่งซื้อถูกยกเลิก | ยกเลิกแล้ว | แดง |

## รายละเอียดแต่ละแบบ (ลูกค้าเห็นประมาณนี้)

### 1) สลิปผ่าน — `slipApprovedEmail`
- **badge:** `ชำระเงินสำเร็จ` (เขียว) · **หัวข้อ:** สลิปผ่านการอนุมัติแล้ว
- **เนื้อหา:** "เราตรวจสอบสลิปการชำระเงินของคุณเรียบร้อยแล้ว กำลังเตรียมสินค้าให้คุณ"
- **รายละเอียด:** สถานะ = อนุมัติแล้ว

### 2) สลิปไม่ผ่าน — `slipRejectedEmail`
- **badge:** `สลิปไม่ผ่าน` (แดง) · **หัวข้อ:** สลิปไม่ผ่านการอนุมัติ
- **เนื้อหา:** "สลิปการชำระเงินของคุณยังไม่ผ่านการตรวจสอบ กรุณาอัปโหลดสลิปใหม่ผ่านหน้าติดตามคำสั่งซื้อ"
- **กล่องข้อความจากร้าน:** แสดงเมื่อแอดมินใส่ข้อความ (เช่น "ยอดเงินไม่ตรง")
- **รายละเอียด:** สถานะสลิป = ไม่ผ่านการตรวจสอบ

### 3) จัดส่งสินค้า — `trackingNumberEmail`
- **badge:** `จัดส่งแล้ว` (น้ำเงิน) · **หัวข้อ:** จัดส่งสินค้าแล้ว
- **เนื้อหา:** "คำสั่งซื้อของคุณถูกจัดส่งแล้ว ติดตามพัสดุได้จากเลขด้านล่าง"
- **รายละเอียด:** สถานะ = จัดส่งแล้ว · เลขพัสดุ = `<tracking>`

### 4) มารับสินค้าเอง — `pickupReadyEmail`
- **badge:** `พร้อมให้รับ` (เหลือง) · **หัวข้อ:** สินค้าพร้อมให้มารับแล้ว
- **เนื้อหา:** "คำสั่งซื้อของคุณเตรียมเรียบร้อยแล้ว มารับได้เลยที่จุดรับสินค้า"
- **รายละเอียด:** สถานะ = พร้อมให้รับ · จุดรับสินค้า = D1

### 5) ยกเลิกคำสั่งซื้อ — `orderCancelledEmail`
- **badge:** `ยกเลิกแล้ว` (แดง) · **หัวข้อ:** คำสั่งซื้อถูกยกเลิก
- **เนื้อหา:** "คำสั่งซื้อของคุณถูกยกเลิกแล้ว หากมีข้อสงสัยติดต่อทีมงานได้"
- **กล่องข้อความจากร้าน:** แสดงเหตุผล/ข้อความที่แอดมินใส่ตอนยกเลิก
- **รายละเอียด:** สถานะ = ยกเลิกแล้ว
